You are studying a small set of research trajectories so that a later agent
can do this kind of question better. You are given:

- a list of trajectories, each with the question, the gold answer, the
  agent's prediction, the F1 of that prediction, and the tool calls the
  agent made;
- optionally, a previous artifact that was produced earlier and not
  accepted, with a note about what to improve.

Produce a specialization artifact. It must be a single JSON object with
exactly these keys, and nothing else:

{
  "query_style_rules":    [ "...", ... ],
  "source_preferences":   { "prefer": [ "..." ], "avoid": [ "..." ] },
  "stopping_heuristic":   "one short sentence",
  "typical_hop_count":    <integer, usually between 2 and 5>,
  "few_shots":            [ { "question": "...", "trajectory": "...", "answer": "..." }, ... ],

}

Guidance:
- Keep each rule short and concrete (5-20 words). Generic advice like
  "be accurate" is not useful.
- `stopping_heuristic` should describe when the agent can stop searching,
  grounded in what the trajectories actually show (e.g. "stop once the
  bridge entity is confirmed on a second page").
- Include at most 3 few-shots. Prefer trajectories whose F1 was high and
  whose pattern is likely to generalize to similar questions.
- `system_prompt` must not exceed about 120 words. It replaces the baseline
  system prompt for the specialized agent.
- Output only the JSON object. No commentary, no markdown fences.
